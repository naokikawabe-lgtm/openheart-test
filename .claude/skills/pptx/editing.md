# PPTX テンプレート編集手順

## 前提知識
PPTXはZIPパッケージで、スライドの構造やテキストがOffice Open XMLで記述されている。
拡張子を `.pptx` から `.zip` に変えると `ppt/slides/slide1.xml` のようにスライド1枚ごとのXMLファイルを確認できる。

## Editing Workflow

### 1. テンプレートの分析
```bash
# テンプレートPPTXを解凍してXML構造を確認
cp template.pptx template.zip
unzip template.zip -d template_extracted/
ls template_extracted/ppt/slides/
```

### 2. スライド構造の把握
- `ppt/slides/slideN.xml` — 各スライドのコンテンツ
- `ppt/slideLayouts/` — レイアウト定義
- `ppt/slideMasters/` — マスタースライド
- `ppt/presentation.xml` — プレゼンテーション全体の設定

### 3. XMLの編集
各スライドのXMLを直接編集する:
- テキストボックスの内容を置換
- 図形の位置・サイズを調整
- 新しいスライドの追加・削除

### 4. PPTXの再パッケージ
```bash
cd template_extracted/
zip -r ../output.pptx . -x ".*"
```

## レイアウトパターン一覧

### 1. カバー（表紙）
- 用途: タイトル・日付・宛先
- 使いどころ: 資料の冒頭

### 2. アジェンダ
- 用途: 目次・全体構成の提示
- 使いどころ: 表紙の直後

### 3. セクションディバイダー
- 用途: 章の切り替え
- 使いどころ: 大テーマの区切り

### 4. タイトル＋本文（1カラム）
- 用途: 主張＋説明テキスト
- 使いどころ: 方針・結論の提示

### 5. タイトル＋2カラム
- 用途: 比較・対比
- 使いどころ: 現状vs提案、技術A vs B

### 6. タイトル＋箇条書き
- 用途: ポイントの列挙
- 使いどころ: 課題一覧、メリット一覧

### 7. タイトル＋表
- 用途: 構造化された情報の提示
- 使いどころ: 機能比較、スケジュール、費用内訳

### 8. タイトル＋図解（プロセス）
- 用途: フロー・手順の可視化
- 使いどころ: 導入プロセス、ワークフロー

### 9. タイトル＋図解（マトリクス）
- 用途: 2軸での分類・ポジショニング
- 使いどころ: 競合マッピング、優先度マトリクス

### 10. タイトル＋グラフ
- 用途: データの視覚化
- 使いどころ: 市場規模推移、KPI実績

### 11. まとめ・Next Steps
- 用途: 結論とアクション
- 使いどころ: 資料の最後

## ビジュアルQA（サブエージェント推奨）

### QAチェック項目
1. **テキスト切れ** — テキストがボックスからはみ出していないか
2. **フォントの折り返し** — 意図しない位置で改行されていないか
3. **レイアウト崩れ** — 要素の位置ずれ、重なりがないか
4. **テンプレートの消し残し** — プレースホルダーテキストが残っていないか
5. **一貫性** — フォントサイズ、色、マージンが統一されているか

### QA手順
1. PPTX → PDF/画像に変換
2. サブエージェントで1枚ずつ画像をチェック
3. 問題を検出したらXMLを修正
4. 再変換して再チェック
5. 全スライドがクリーンになるまで繰り返す
