# Decisions - 意思決定の記録

## COO/CFO/CSOの仕事リスト：フォーカスと責任の明確化

### 会社プロフィール
- **会社名**: OpenHeart
- **事業**: SaaS / ソフトウェア
- **フェーズ**: 0→1（立ち上げ期・PMF模索中）
- **チーム規模**: 1〜5人
- **役職**: COO/CFO/CSO（開発以外の全てを管掌）

### 2026-03-19（初回設定）

#### テーマ
0→1フェーズのCOO/CFO/CSOとして「自分が握るべき仕事」と「手放す・後回しにする仕事」の線引き

---

#### 【毎日見る！自分の仕事】
1. **PMF検証の推進** — 顧客の声を直接拾い、プロダクトが刺さるかの仮説検証を回し続けること（開発チームへのフィードバックも含む）
2. **売上をつくる** — 初期顧客の獲得・商談・クロージングを自らやり、再現可能な売り方の型を見つけること
3. **キャッシュの管理** — ランウェイ（資金の残り期間）を常に把握し、資金調達や収支計画を主導すること
4. **Go-to-Market戦略の設計** — 誰に・何を・どう届けるかの戦略を決め、実行の優先順位を自分で判断すること
5. **経営の意思決定と優先順位づけ** — 開発以外の全領域で「今やること・やらないこと」を決め、リソース配分を最適化すること

#### 【やらないこと・手放すこと】
- **プロダクト開発の実作業** → 開発チームに任せる（方向性のインプットに徹する）
- **完璧な資料づくり** → 0→1では速度が命。80点で出して反応を見る
- **仕組み化・制度設計の作り込み** → PMFが見えるまでは後回し。評価制度・マニュアル整備は時期尚早
- **全ての問い合わせ対応** → 対応パターンが見えたらFAQ化・テンプレ化してメンバーに渡す
- **SNS運用・コンテンツ作成の実作業** → 戦略だけ決め、実行は外注 or メンバーに任せる

---

#### 運用ルール
- Claudeはこのリストからズレた行動を検知したら指摘する
- 特に「仕組み化の作り込み」「完璧な資料づくり」に時間を使い始めたら即座に止める
- 「それは今やるべきですか？PMFの前にやる意味がありますか？」と問いかける
- 定期的にリマインドする
- 毎週金曜日に振り返りを実施する
